# Scientific Positioning

## What MusicInteractionFramework is not

MusicInteractionFramework is not:

- a new temporal logic
- a replacement for Allen interval algebra
- a new process calculus
- a MIDI library
- a music information retrieval library

## What MusicInteractionFramework is

MusicInteractionFramework is a meta-framework for representing, implementing, comparing and evaluating temporal interaction models.

It provides:

- a common ontology
- a graph-based representation layer
- a pipeline architecture
- a plugin architecture
- a metric infrastructure
- reproducibility through provenance

## Relation to existing temporal theories

Existing temporal theories can be implemented as plugins or interaction models.

Examples:

- Allen interval algebra can produce temporal relations.
- Temporal logic can define constraints on sequences.
- Event calculus can describe state changes.
- Petri nets or timed automata can model processes.
- Conversation analysis can define turn-taking and overlap models.

## Original contribution

The original contribution is not one new theory of time.

The contribution is the integration layer: a software and conceptual framework in which multiple temporal interaction theories can be expressed, compared and operationalized on shared session data.
